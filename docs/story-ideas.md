# Story Ideas

Potential topics for new blog stories. Each should follow the format: short, real-world, from-the-trenches — a clear idea without boring the audience.

## Candidates

### Offline-first architecture [NEXT — draft after April 22]
Challenges and patterns of building a strongly offline-first mobile app. Data sync strategies, conflict resolution, local-first persistence. Keep it generic enough to not spoil the current project. Connected to the KMP offline-first tooling choice — internal STRV mobile meeting talk on 2026-04-22. Write the public-friendly version after that talk.

### KMP in practice
Lessons from the TechnoAlpine snowmaking project — adopting Kotlin Multiplatform in a real production app. What worked, what didn't, and where KMP shines vs where it's still rough.

### Building a reusable native SDK
The MLS (Major League Soccer) experience — designing a native SDK that provides backend functionality and is reusable beyond a single app. API design decisions, versioning, and separation of concerns.

### Convention plugins beyond basics
Going beyond standard gradle plugins — building custom convention plugins for project-specific needs. The PoEditor story is already told, but there's more to the build-logic architecture in multi-module Android projects.

### Legacy rewrite patterns
What you learn from inheriting and modernizing codebases at an agency. When to rewrite vs refactor, how to convince stakeholders, and patterns that make the transition smoother.

### AI-assisted development in practice
Real experience of integrating agentic AI coding into the Android development workflow. What project documentation matters, how tests become the safety net, and the craftsman's evolving role.
